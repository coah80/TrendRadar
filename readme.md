### **Smart push strategies**

Three push modes:

| Mode | Best for | When it pushes | What is delivered | Typical use case |
|------|----------|----------------|-------------------|------------------|
| **Daily summary** (`daily`) | Managers / general users | Scheduled pushes (default: once per hour) | All matched news for the day + a "new items" section | Daily summaries; comprehensive day overview |
| **Current leaderboard** (`current`) | Content creators / media | Scheduled pushes (default: once per hour) | Matches from the current leaderboards + new items section | Real-time hotspot tracking; spot what's trending now |
| **Incremental monitoring** (`incremental`) | Investors / traders | Only when there are new matches | Newly appeared news containing matched frequency words | Avoid duplicate notifications; high-frequency monitoring |

Optional: push time window control (can be combined with any mode):

- Time window limits: set a time range for pushes (e.g., 09:00-18:00 or 20:00-22:00)
- Push frequency control:
  - Multiple pushes within the window: send a push each execution while in the window
  - Once per day: only send one push while inside the window (good for daily summary or current leaderboard)
- Example scenarios:
  - Push only during work hours 09:00-18:00
  - Receive a consolidated summary at night (e.g., 20:00-22:00)
  - Avoid being disturbed outside working hours

Tip: this feature is disabled by default. Enable it by setting `push_window.enabled` in `config/config.yaml`.

<details id="frequencywordstxt-configuration-guide">
<summary><strong>ğŸ‘‰ Click to expand: frequency_words.txt configuration guide</strong></summary>
<br>

Configure monitoring keywords in `frequency_words.txt`. The file supports three syntaxes and grouping.

Keywords listed earlier have higher priority â€” you can reorder them to reflect importance.

| Syntax | Symbol | Purpose | Example | Matching rule |
|--------|--------|---------|---------|---------------|
| Plain word | (none) | Basic match | `Huawei` | Match if any word appears |
| Required word | `+` | Narrow scope | `+phone` | Must be present in addition to plain keywords |
| Exclusion | `!` | Exclude noise | `!ad` | If present, exclude the item |

### Basic syntax

#### 1. Plain keywords
```txt
Huawei
OPPO
Apple
```
Matches if the title contains any one of these words.

#### 2. Required words `+word`
```txt
Huawei
OPPO
+phone
```
Matches only if both a plain keyword and the required word appear.

#### 3. Exclusion `!word`
```txt
Apple
Huawei
!fruit
!price
```
If a filtered word appears, the news is excluded even if it contains a keyword.

### Grouping â€” separate groups with blank lines

Core rule: Use a blank line to separate keyword groups. Each group is counted independently.

Example configuration:

```txt

Huawei
OPPO
+release

A-shares
Shanghai
Shenzhen
+rise-fall
!prediction

World Cup
Euro Cup
Asia Cup
+match
```

### **Personalized hotness algorithm**

TrendRadar reorders cross-platform trending items using a custom weighting algorithm rather than relying on any single platform's ranking:

- Rank importance (60% by default): items with high rank on platforms weigh more
- Frequency (30%): items that appear repeatedly across grabs weigh more
- Rank quality (10%): considers both frequency and high placement

> The combined ranking can be adjusted to suit different use cases.

<details>
<summary><strong>ğŸ‘‰ Click to expand: weight adjustment</strong></summary>
<br>

Default config is balanced. Two sample scenarios:

Realtime hotspot focus:
```yaml
weight:
  rank_weight: 0.8    # prioritize rank
  frequency_weight: 0.1
  hotness_weight: 0.1
```
Best for content creators and marketers who want the currently hottest topics.

Depth-topic focus:
```yaml
weight:
  rank_weight: 0.4
  frequency_weight: 0.5  # emphasize persistence
  hotness_weight: 0.1
```
Best for investors, researchers, and journalists who need deeper trend analysis.

Adjustment rules:
1. The three numbers must sum to 1.0
2. Increase the factor that matters more to you
3. Change gradually (0.1-0.2) and observe results
